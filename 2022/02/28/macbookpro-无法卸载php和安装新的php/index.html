Posted by wuerqi on 2022-02-28

macbookpro 无法卸载php和安装新的php

背景:
本人在macbookpro安装phpcs-pre-commit-hook后,在进行git commit时,composer.json报错:

Running Code Sniffer. Code standard PSR12.
PHP Fatal error: Composer detected issues in your platform: Your Composer dependencies require a PHP version “>= 7.4.1”. You are running 7.3.28. in />usr/local/var/www/dnmp/www/purchase/vendor/composer/platform_check.php on line 24

突然想起项目是安装在容器中的,本地php版本是7.3.8导致的这个错误,故需要将本地php升级,在度娘上搜索到使用brew uninstall php@7.3可以卸载php,开始踩坑之旅.

php安装7.4

首先查看本地php版本

brew list
得到本地php版本php@7.3

卸载成功!
在进行php安装7.4的过程中,又遇到新的问题:

执行安装php7.4时报错:

继续度娘,终于找到一个靠谱的解释:
原文: https://zhuanlan.zhihu.com/p/383707713

终于安装php@7.4成功.

brew services start php@7.4
启动成功

使用php查看当前版本,报错:

-bash: /usr/local/opt/php@7.3/bin/php: No such file or directory

是因为macbookpro的快捷命令行导致的

得到

source ~/.bash_profile

pecl install redis

```

结束!
